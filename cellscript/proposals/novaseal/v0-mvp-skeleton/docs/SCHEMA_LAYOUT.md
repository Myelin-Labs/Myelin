# NovaSeal v0 Schema Layout

**Date**: 2026-05-31
**Extractor**: `scripts/novaseal_schema_layout.py`
**Report**: `target/novaseal-schema-layout.json`
**Encoding profile**: `packed-fixed-v0-reference`

This slice turns the three human-readable `.schema` files into a machine-readable fixed layout. It is deliberately not a full Molecule compiler output.

## Important Limitations

This layout is not full production encoding by itself:

- canonical packed-reference vectors exist, but they are not Molecule output,
- fixed-width wallet signing vectors now exist in
  `target/novaseal-wallet-signing-vectors.json` and are generated by
  `/home/arthur/a19q3/CellScript/scripts/novaseal_wallet_signing_vectors.py`,
- no Molecule table/schema file is generated,
- no comparison is made against CellScript compiler ABI metadata,
- no real BTC wallet/PSBT signing implementation consumes the output,
- CKB VM and resolved transaction harnesses consume compatible packed data, but
  this is still not a published wallet/Molecule reference implementation.

Production still requires public/shared CellDep attestation, external BIP340 TCB
review, and real wallet/PSBT integration.

The report is still useful because it freezes the first mechanically testable interpretation of field order, widths, offsets, and source hashes.

The next layer is `target/novaseal-canonical-vectors.json`, generated by `scripts/novaseal_canonical_vectors.py`. Those vectors are packed-reference test bytes, still not Molecule output.

## Fiber/xUDT Note

`NovaSealCellV0` currently has no `amount` field. The schema comment about an xUDT-compatible fungible profile remains a future profile note, not an implemented layout.

The extractor therefore reports:

```text
fiber_fungible_profile.status = not_defined_in_v0_layout
fiber_fungible_profile.amount_offset = null
```

Do not claim Fiber admission evidence until a concrete fungible profile file defines the amount field and its offset.
